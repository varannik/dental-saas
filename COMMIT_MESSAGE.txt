chore(dev): harden local service orchestration and gateway smoke flow

Make local full-stack verification deterministic by improving migration/seed/start/smoke scripts,
stabilizing gateway upstream resolution, and fixing tenant creation behavior used by gateway tests.

## What Changed

- Strengthened database automation:
  - Updated `scripts/database/migrate.sh` to run real Drizzle migrations, verify tables, and apply SQL fallback when schema is missing.
  - Updated `scripts/database/seed.sh` to ensure required local tenant data exists idempotently and clean up integration test artifacts.
- Added and wired gateway smoke coverage:
  - Added `scripts/test/gateway-smoke-checks.sh` to run register/login/me/create-tenant flow through API gateway.
  - Added `make gateway-smoke-checks` target in `Makefile`.
  - Enhanced smoke failure output to print response bodies for faster debugging.
- Improved local service startup reliability:
  - Updated `scripts/dev/start-services.sh` to validate Docker/Postgres/Redis readiness, free conflicting ports, start auth/users/gateway with logs and pid files, and run health checks.
  - Passed explicit upstream env vars to gateway startup to avoid ambiguous local host resolution.
- Stabilized gateway proxy/runtime defaults:
  - Updated auth/users proxy defaults in `apps/api-gateway/src/routes/auth.proxy.ts` and `apps/api-gateway/src/routes/users.proxy.ts` to loopback-safe upstreams.
  - Updated Redis fallback in `apps/api-gateway/src/plugins/rate-limit.ts` for consistent local connectivity.
  - Updated gateway tests in `apps/api-gateway/src/__tests__/index.test.ts` for tenant proxy behavior.
- Fixed users tenant creation path used by smoke checks:
  - Updated `services/users/src/services/tenant.service.ts` to serialize locale/language arrays safely for DB insert values.
- Refined local infra wiring:
  - Updated `scripts/lib/docker.sh` and `infrastructure/docker/docker-compose.yml` for improved local dependency readiness and consistency.
  - Added `services/users/Dockerfile` and aligned compose/service config usage.

## Validation

- `pnpm --filter @saas/api-gateway typecheck`

- `make start-services`
- `make gateway-smoke-checks` (full flow passed)
