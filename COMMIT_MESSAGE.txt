feat(clinical): patients API, gateway proxy, Docker, and gateway smoke coverage

Introduce the @saas/clinical service (tenant-scoped patients CRUD, list/search, history, soft delete)
behind JWT, wire the API gateway to forward /api/v1/patients* to the clinical service, add a compose
service and local dev startup, and extend gateway smoke checks with a clear pass summary and optional
verbose JSON. Phase 2 status doc (`status/08-…`) updated to match.

## What changed

- **Clinical service** (`services/clinical/`):
  - Fastify app on port 4003, routes under `/patients`, Drizzle via `@saas/config`, audit hooks, Vitest.
  - `tsconfig` uses `rootDir: "src"` and package imports from `@saas/config` so `dist/index.js` is the
    runtime entry.
  - **Dockerfile**: build `@saas/config` then clinical; runtime copies `packages/config` so
    `@saas/config` resolves in the image.
- **API gateway** (`apps/api-gateway/src/routes/patients.proxy.ts`):
  - Proxies `GET|POST|… /api/v1/patients` and `/patients/*` to `CLINICAL_SERVICE_URL` (default
    `http://127.0.0.1:4003`), forwarding auth, optional `Content-Type`, and `x-request-id`.
  - Tests (`index.test.ts`): patients route exercises real proxy behavior via `fetch` mock; upstream
    URL expectations aligned with `127.0.0.1` defaults.
- **Docker Compose** (`infrastructure/docker/docker-compose.yml`): `clinical` service, image
  `dental-saas-clinical:local`, port 4003, depends on healthy Postgres.
- **Local dev** (`scripts/dev/start-services.sh`): starts clinical on 4003 with `CLINICAL_SERVICE_URL`
  passed into the gateway; stops `dental-saas-clinical` container when present to avoid port clashes.
- **Gateway smoke** (`scripts/test/gateway-smoke-checks.sh`): patient flows through the gateway (create,
  list, get, history, delete), assertion checklist + correlation line; documents upstream URLs and
  `.run/logs` vs Docker; `SMOKE_VERBOSE=1` dumps full JSON bodies.
- **Makefile**: help strings for `start-services` and `gateway-smoke-checks`.

## Validation

- `pnpm --filter @saas/config build && pnpm --filter @saas/clinical build`
- `pnpm --filter @saas/clinical test`
- `pnpm --filter @saas/api-gateway exec vitest run`
- With Postgres/Redis and `make start-services`: `make gateway-smoke-checks`
- Optional: `docker compose build clinical` and run container against `.env`
