feat(api-gateway): scaffold gateway app with proxy and security plugins

Add the API Gateway app for Week 4 with Fastify-based proxy routes, shared middleware/plugins,
and a comprehensive gateway test suite to validate auth, tenant resolution, rate limiting, and routing behavior.

## What Changed

- Created new workspace app `@saas/api-gateway`:
  - Added `apps/api-gateway/package.json` with scripts for `dev`, `build`, `start`, `typecheck`, and `test`.
  - Added `apps/api-gateway/tsconfig.json` and updated app README.
- Implemented gateway server entry and registration flow:
  - Added `src/index.ts` with plugin/middleware/route registration.
  - Added service config defaults (`GATEWAY_HOST`, `GATEWAY_PORT`) and `/health` endpoint.
  - Added centralized error handler returning `{ error: string }` payloads.
- Added middleware and plugins:
  - `middleware/request-id.ts` for request ID propagation.
  - `middleware/logging.ts` for response logging.
  - `middleware/tenant-resolver.ts` for JWT claim extraction (`tenantId`, `userId`, `roles`).
  - `plugins/auth.ts` for JWT authorization checks.
  - `plugins/rate-limit.ts` with Redis key pattern `dental:ratelimit:{tenant}:{user}:{endpoint}` and role-based limits.
  - `plugins/cors.ts` and `plugins/websocket.ts`.
  - Wrapped shared plugins/middleware using `fastify-plugin` to avoid encapsulation visibility issues.
- Added proxy and placeholder routes:
  - `routes/auth.proxy.ts` -> forwards `/api/v1/auth/*` to auth service.
  - `routes/users.proxy.ts` -> forwards `/api/v1/users/*` to users service (auth-protected).
  - `routes/patients.proxy.ts` placeholder with auth + 501 response.
  - `routes/voice.proxy.ts` websocket placeholder route.
  - Fixed proxy path rewriting to strip `/api/v1/{service}` prefixes correctly.
- Added comprehensive gateway tests under `apps/api-gateway/src/__tests__`:
  - `index.test.ts` for health, auth protection, proxy forwarding, upstream error contract, CORS route registration, websocket route registration.
  - `auth-plugin.test.ts` for missing/valid token behavior.
  - `tenant-resolver.test.ts` for JWT claim extraction behavior.
  - `rate-limit-plugin.test.ts` for `429` enforcement and Redis-unavailable fallback behavior.

## Validation

- `pnpm --filter @saas/api-gateway test`
- `pnpm --filter @saas/api-gateway typecheck`
- `pnpm --filter @saas/api-gateway build`
