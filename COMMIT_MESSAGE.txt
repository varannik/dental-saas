fix(gateway): proxy headers and users path; extend smoke CRUD; migration ledger check

Align gateway forwarding with Fastify upstream expectations, exercise full user CRUD through the gateway
in smoke checks, verify Drizzle’s migration ledger in the correct schema, and keep users service unit
tests green after tenant inserts started using `sql` fragments.

## What changed

- API gateway proxies (`apps/api-gateway/src/routes/auth.proxy.ts`, `users.proxy.ts`):
  - Forward `Content-Type` only when the client sent it, so body-less methods (for example `DELETE`)
    are not forced to `application/json` and no longer trigger empty-body JSON parse errors upstream.
  - Register `GET|POST|… /api/v1/users` on `/users` in addition to `/users/*`, so the collection route
    is proxied (previously only `/users/...` matched).
- Gateway smoke script (`scripts/test/gateway-smoke-checks.sh`):
  - After register/login/me/create-tenant, run user CRUD through the gateway: create, list (assert id
    present), get by id, patch, delete; add `http_patch_json` / `http_delete_auth`; widen DB cleanup
    to remove both the auth smoke user and the CRUD user email.
- Local migration verification (`scripts/database/migrate.sh`):
  - Check `drizzle.__drizzle_migrations` (not `public`), optionally report ledger row count, and keep
    table-count checks as fallback messaging.
- Users tests (`services/users/src/__tests__/tenant.service.test.ts`):
  - Mock `sql` on `drizzle-orm` so `createTenant` unit tests match the real service.

## Validation

- `pnpm --filter @saas/users exec vitest run`
- `pnpm --filter @saas/api-gateway exec vitest run` (if you touch gateway tests in the same change)
- `make gateway-smoke-checks` (with Docker Postgres/Redis, migrations applied, and services running)
