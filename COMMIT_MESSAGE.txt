feat(config): add complete Drizzle schema modules from YAML

Implement the Phase 1 database foundation by creating a full Drizzle schema layer
in `@dental/config`, aligned to `schema-core.yaml` and `schema-agent-extensions.yaml`.

## What Changed

- Added modular schema definitions under `packages/config/src/schema/`:
  - `tenancy.ts` (tenants, users, sessions, RBAC, audit)
  - `clinical.ts` (patients, encounters, clinical notes)
  - `imaging.ts` (imaging studies/objects, AI models, jobs, predictions)
  - `billing.ts` (treatment plans, claim headers/lines)
  - `voice.ts` (voice sessions, utterances, recordings)
  - `agent.ts` (workflows, executions, tools, HITL, memory, metrics, undo history)
- Added `packages/config/src/schema/index.ts` barrel for a clean schema export surface.
- Updated `packages/config/src/index.ts` to re-export all schema modules.
- Preserved multilingual support fields (locale/language) for UI, voice, and agent flows.
- Added production-ready indexes, composite keys, and foreign key relationships.
