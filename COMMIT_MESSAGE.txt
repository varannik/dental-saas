feat(docs,tooling): add multilingual architecture support and restore ESLint setup

Introduce end-to-end language/locale support in schema and architecture docs
for Persian-ready voice workflows, and add a working ESLint flat config for
TypeScript linting in pre-commit.

## What Changed

- Added tenant-level language policy fields in core schema:
  - `default_locale`
  - `supported_locales`
  - `supported_languages`
- Added user/patient locale + language preference fields for UI and voice routing.
- Added voice session locale/routing fields:
  - `input_locale`, `output_locale`
  - `asr_language`, `tts_voice_id`
- Added utterance-level language metadata:
  - `transcript_locale`
  - `normalized_transcript`
- Added language-aware agent memory metadata and index in agent extensions:
  - `content_locale`, `content_language`
  - `idx_agent_memories_tenant_lang_scope`
- Updated architecture/status docs to include:
  - Persian (`fa-IR`) compatibility requirements
  - RTL UI expectations
  - language-aware RAG retrieval rules
  - locale-specific test coverage and remaining implementation gaps
