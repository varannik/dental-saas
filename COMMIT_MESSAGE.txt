chore(dev-env): add env template and align local config defaults

Standardize local development configuration by adding a complete `.env.example`,
fixing PostgreSQL password mismatch in local startup output, and aligning
Terraform region validation defaults with the project region strategy.

## What Changed

- Added `.env.example` with required variables for:
  - app/runtime
  - database/redis
  - auth/security
  - AI/voice
  - storage (S3/MinIO)
  - AWS/LocalStack
  - observability and testing
- Fixed DB credential mismatch in `scripts/local/start.sh`:
  - replaced incorrect `postgres_dev_password` reference
  - now logs connection using `postgres:postgres` (matching local Docker defaults)
- Fixed region inconsistency in `scripts/terraform/validate-region.sh`:
  - default expected region now uses `AWS_REGION` fallback to `eu-central-1`
  - local validation uses the same expected region
  - bucket-region fallback checks now use expected region instead of hardcoded `us-east-1`
