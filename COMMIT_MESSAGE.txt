test(auth): consolidate auth tests and add full service coverage

Move auth testing to the service-local test suite and replace placeholder coverage with
route/unit/integration/contract checks so auth behavior is validated end-to-end in one place.

## What Changed

- Consolidated auth tests into `services/auth/src/__tests__`:
  - Removed duplicated top-level auth integration scaffolding under `tests/integration`.
  - Cleaned empty integration subdirectories and updated integration docs ownership guidance.
- Added comprehensive auth route tests:
  - `routes/register.test.ts`
  - `routes/login.test.ts`
  - `routes/logout.test.ts`
  - `routes/refresh.test.ts`
  - `routes/me.test.ts`
  - `routes/sessions.test.ts`
  - Covers success and validation/error-path behavior for request handling.
- Added core unit tests:
  - `token.service.test.ts` for issue/verify/refresh behavior and invalid-token rejection.
  - `middleware/authenticate.test.ts` for bearer header parsing edge cases.
  - `auth.service.test.ts` for logout delegation behavior.
- Added service-level integration and contract suites:
  - `integration/auth.integration.test.ts` for register/login/logout/me lifecycle checks.
  - `integration/auth.contract.test.ts` for response-shape and key field contract assertions.
- Extended auth test scripts:
  - Added `test:integration`, `test:contract`, and `test:all` in `services/auth/package.json`.
