feat(users): add user service APIs with comprehensive tests

Create Week 4 User Service with Fastify routes for users, tenants, locations, and roles,
plus schema, service, and server-level test coverage to validate core behavior.

## What Changed

- Scaffoled new workspace service `@saas/users`:
  - Added `services/users/package.json` with scripts for `dev`, `build`, `start`, `typecheck`, and `test`.
  - Added `services/users/tsconfig.json`.
  - Updated `services/users/README.md` with run/test commands.
- Implemented users service server entrypoint:
  - Added `services/users/src/index.ts` with:
    - `getDefaultUsersServiceConfig()` using `USERS_HOST`/`USERS_PORT` defaults.
    - `/health` endpoint.
    - central error handler for validation/runtime errors.
    - route registration via `registerUserServiceRoutes`.
- Added route modules under `services/users/src/routes`:
  - `users.ts` for user CRUD.
  - `tenants.ts` for tenant list/get/create.
  - `locations.ts` for location list/create/delete.
  - `roles.ts` for role list/create/delete.
  - `index.ts` to register all route groups.
- Added request validation schemas under `services/users/src/schemas`:
  - `user.schema.ts`, `tenant.schema.ts`, `location.schema.ts`, `role.schema.ts`.
- Added service layer modules under `services/users/src/services`:
  - `user.service.ts` (list/get/create/update/delete, duplicate email guard, optional user-tenant link).
  - `tenant.service.ts` (list/create/get).
  - `location.service.ts` (list/create/delete).
  - `role.service.ts` (list/create/delete, permission-key mapping to `role_permissions`).
- Expanded test suite under `services/users/src/__tests__`:
  - `schemas.test.ts` for schema-level validation/defaults.
  - `routes.users.test.ts`, `routes.tenants.test.ts`, `routes.locations.test.ts`, `routes.roles.test.ts` for route behavior and status codes.
  - `user.service.test.ts`, `tenant.service.test.ts`, `location.service.test.ts`, `role.service.test.ts` for service logic with mocked DB/Drizzle dependencies.
  - `index.test.ts` for `/health` and server-level error contract behavior.

## Validation

- `pnpm --filter @saas/users test`
- `pnpm --filter @saas/users typecheck`
- `pnpm --filter @saas/users build`
