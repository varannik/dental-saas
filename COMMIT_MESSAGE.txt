feat(web): Next.js practice shell + gateway CORS, auth proxy, and Redis ergonomics

Introduce `apps/web` (`@saas/web`) as the Phase 2 dental-core front door on the monorepo
(`@saas/*`), and fix API-gateway auth forwarding and local cross-origin behavior.

Web app (`apps/web`)

- Next.js 15 (App Router, `src/`, Tailwind v4, `@/*` alias), shared UI primitives (card, button,
  input, label, table) aligned with shadcn-style patterns.
- Root layout: Inter + JetBrains Mono, TanStack Query `Providers`, clinical navy / teal tokens in
  `globals.css`.
- Public marketing `/`; auth routes `/login` and `/register` with gateway calls to
  `POST /api/v1/auth/login` and `POST /api/v1/auth/register` using demo tenant id
  `11111111-1111-4111-8111-111111111111`.
- Dashboard group behind `DashboardShell`: collapsible sidebar, header, breadcrumbs, client-side
  session gate + `?next=` redirect after login; routes `/dashboard`, `/dashboard/schedule`,
  `/dashboard/reports`, `/patients`, `/patients/[id]`, encounters placeholder,
  `/settings`.
- Data layer: `apiFetch` / `getApiBaseUrl()` (`NEXT_PUBLIC_API_GATEWAY_URL`, default
  `http://localhost:4000`), auth token in `localStorage`, Zustand auth + encounter stores,
  hooks for patients list, patient detail, and dental chart queries.
- Patient list (search, pagination, quick-add `POST /api/v1/patients`); patient detail with
  `ChartSection` odontogram (universal 32-tooth layout) driven by `GET .../:id/chart`; voice and
  notes stubs.
- Package metadata, `transpilePackages` for `@saas/types`, and README run instructions.

API gateway

- **Auth proxy:** Build upstream URLs like users/patients proxies by stripping `/api/v1` from
  `request.url` (with query stripped). Fixes `/auth/register` becoming `/auth/auth/register` and
  upstream **404** when Fastify exposes prefixed paths as `/auth/...`.
- **CORS:** When `CORS_ORIGINS` is unset, use reflected `origin: true` in non-production so common
  dev hosts (LAN IP, alternate ports) receive `Access-Control-Allow-Origin`; explicit
  `CORS_ORIGINS` still overrides; production without env keeps a conservative fallback list.
- **Rate limit:** `redis.on?.('error', ...)` so connection errors are logged instead of unhandled
  `error` events when Redis is down (enforcement already degrades in the guard).

Web client

- **`getApiBaseUrl()`:** Strip accidental `/api` or `/api/v1` suffixes from
  `NEXT_PUBLIC_API_GATEWAY_URL` so requests are not doubled as `/api/v1/api/v1/...`.

## Validation

- `pnpm install` from repo root; start Postgres + Redis (e.g. `make docker-up` as needed).
- `pnpm --filter @saas/api-gateway test`
- `pnpm --filter @saas/web lint && pnpm --filter @saas/web typecheck && pnpm --filter @saas/web build`
- Local smoke: gateway `GET /health`; auth on `:4001`; register + login from `/register` and
  `/login` against `http://localhost:4000/api/v1/auth/*`.
