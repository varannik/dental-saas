feat(dev): graceful start-services teardown; chore(auth): error handler + integration diagnostics

Improve `make start-services` by detecting **TCP LISTEN** sockets per dedicated port before
bringing up auth (4001), users (4002), clinical (4003), and the API gateway (4000). Busy ports are
stopped with **SIGTERM** first, polled for up to roughly twenty seconds on a half-second cadence,
then **SIGKILL** only if listeners remain. Clear stale launcher **`.run/*.pid`** files when
reconciling ports so restarted runs line up with the actual processes.

Register a Fastify **`registerAuthErrorHandler`** on the auth service (`services/auth/src/lib/error-response.ts`):
**Zod** validation failures respond with **400** and flattened details; other errors reuse
**`statusCode`** when present; non‑production payloads include **`error.cause`** chain detail for faster
debugging while production masks the message behind a generic internal error shape.

Improve auth **integration test helpers**: **`assertHttpStatus`** attaches the parsed response body when
expectations fail (so Drizzle‑style failures and validation bodies surface in Vitest output), and
**`assertIntegrationDatabaseReady`** verifies **`public.users`** exists before fixtures run with a clear
migrate hint and **`TEST_DATABASE_URL`** note. Contract and integration specs call the DB guard in **`beforeAll`**
and use **`assertHttpStatus`** for endpoint checks.

## What changed

- **`scripts/dev/start-services.sh`**: **`port_is_listening`**, **`listeners_on_port`**, **`stop_existing_listener_if_any`**, reconcile step messaging, PID cleanup.
- **`services/auth/src/lib/error-response.ts`**: new shared error handler helpers.
- **`services/auth/src/index.ts`**: register error handler during server build.
- **`services/auth/src/__tests__/integration/helpers.ts`**: **`assertHttpStatus`**, **`assertIntegrationDatabaseReady`**.
- **`services/auth/src/__tests__/integration/auth.contract.test.ts`** and **`auth.integration.test.ts`**: **`beforeAll`** DB check and **`assertHttpStatus`** assertions.
