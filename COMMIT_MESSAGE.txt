feat(types): add shared domain type modules and config package entrypoint

Expand the shared typing layer for Phase 1 by adding schema-derived domain type
modules and exposing a single public export surface for `@dental/config`.

## What Changed

- Added `packages/config/src/index.ts`:
  - re-exports typed env APIs (`env`, `getEnv`, `loadEnv`, `Env`)
  - re-exports DB APIs (`createDatabasePool`, `createDatabaseConnection`, `getDatabase`, `closeDatabase`)
  - keeps NodeNext-safe `.js` re-export paths
- Added schema-derived domain type modules under `packages/types/src/`:
  - `tenancy.ts` (tenant/user/membership/role/access context)
  - `clinical.ts` (patient/encounter/clinical note shapes)
  - `imaging.ts` (study/object/AI prediction shapes)
  - `voice.ts` (session/utterance/recording shapes)
  - `agent.ts` (workflow/execution/step shapes)
  - `billing.ts` (treatment plan/claim/claim line shapes)
- Standardized reusable primitives and literal unions:
  - UUID and ISODateTime aliases
  - const-array enum patterns for strong type safety across modules
