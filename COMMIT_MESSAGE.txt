feat(auth): scaffold auth service and local Docker runtime

Introduce the new `@saas/auth` workspace service with initial API routes, token/session flows,
and local container wiring so authentication can run as part of the platform stack.

## What Changed

- Added new `@saas/auth` package:
  - Created `services/auth/package.json` with build/dev/lint/typecheck/test scripts.
  - Added service TypeScript config (`services/auth/tsconfig.json`).
  - Added app entrypoint bootstrap in `services/auth/src/index.ts`.
- Implemented initial auth API surface:
  - Added route registration and handlers for:
    - `POST /auth/register`
    - `POST /auth/login`
    - `POST /auth/logout`
    - `POST /auth/refresh`
    - `GET /auth/me`
    - `GET /auth/sessions`
  - Added Zod request schemas for login/register/refresh payloads.
- Added core auth/session/token services:
  - Implemented password-based register/login flows with Drizzle queries and bcrypt hashing.
  - Added JWT access/refresh token issue + refresh helpers.
  - Added session creation/revocation/listing using Postgres (`sessions` table) and Redis cache.
- Added middleware utilities and service scaffolding:
  - Added bearer token parsing, permission helper, and tenant context helpers.
  - Added initial placeholder test files for auth services and key routes.
- Added containerization and local stack integration:
  - Added multi-stage Dockerfile for auth service (`services/auth/Dockerfile`).
  - Updated local compose to build and run `auth` service on port `4001` with DB/Redis dependencies.
  - Added root `.dockerignore` to reduce Docker build context and exclude local env files.

## Validation

- Validation commands were not run as part of this message rewrite.
