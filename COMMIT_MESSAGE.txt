feat(phase2): dental chart, treatment plans, CDT seed, gateway routes; fix auth integration build

Add Phase 2 dental-core APIs and reference seed data, wire the API gateway, and harden auth
integration tests plus Docker-friendly TypeScript builds.

Clinical service

- Dental chart: `dental_chart_entries` and history events, schemas, service, and routes under
  `/patients/:id/chart`, `/patients/:id/chart/entries`, `/chart-entries/:id`.
- Treatment plans: workflow-oriented `treatment_plans` / `treatment_plan_items` (billing schema),
  nested routes on patients, top-level `/treatment-plans/:id` for items and present/accept.
- Integration: end-to-end workflow test (patient → encounter → chart → note → plan + item) using
  Fastify inject, shared helpers, and `test:integration`; devDependency `pg` for DB setup.
- Unit coverage: chart-entries, treatment-plans, dental-chart/treatment-plan schemas, extended
  patients routes tests.

Config / database

- CDT seed: illustrative ADA_CDT rows in `code_systems`, `code_values`, and `procedure_catalog` per
  active tenant (`seed-data/cdt-procedures.ts`, `seed/cdt-reference.ts`), invoked from `seed.ts`;
  exported helpers and constants from `@saas/config`.
- Migrations: Drizzle journal and snapshots for chart and treatment-plan workflow (0002, 0003).

API gateway

- Register chart-entries and treatment-plans proxies; extend gateway tests accordingly.

Auth service

- Integration and contract suites use in-process `buildServer()` and `AuthInjectClient` (`app.inject`)
  so Postgres and Redis match the Vitest worker (avoids `Failed query` from a different DB than
  localhost:4001).
- `vitest.config.ts` sets default `DATABASE_URL`, `JWT_SECRET`, and `REDIS_URL` for tests.
- `tsconfig.build.json` excludes `src/__tests__` from `tsc`; `package.json` `build` uses it so Docker
  images do not compile tests or require test-only module resolution.
- Helpers JSON-stringify inject payloads for Fastify `InjectOptions`; optional `AuthTestClient` remains
  for HTTP debugging; `afterAll` dynamically imports `@saas/config` for `closeDatabase()`.
- Integration README updated for the new flow.
