feat(scripts): implement database and redis console scripts, improve local dev workflow

Implement previously stubbed console scripts and modify local startup to skip
unnecessary Terraform/AWS dependencies for local development.

## Issues Resolved

1. **Missing Script Implementations**
   - Database console script was placeholder (not functional)
   - Redis console script was placeholder (not functional)
   - Blocked developers from accessing local services

2. **Unnecessary AWS Dependencies for Local Development**
   - `make local` attempted to run Terraform and create VPC
   - Required AWS credentials unnecessarily
   - Failed with AWS authentication errors
   - LocalStack issues prevented smooth local workflow

## Changes Made

### Script Implementations

**scripts/database/console.sh**
- Implemented PostgreSQL console access for all environments
- Environment validation (local/staging/production)
- Container status checking before connection
- Security checks and warnings for production access
- Helpful usage tips (psql commands, how to exit, etc.)
- Proper error handling with informative messages

**scripts/redis/console.sh**
- Implemented Redis CLI console access for all environments
- Environment validation (local/staging/production)
- Container status checking before connection
- Security warnings for production access
- Redis command tips (PING, KEYS, INFO, etc.)
- Interactive console with helpful guidance

**scripts/local/start.sh**
- Modified to skip Terraform step for local development
- Removed unnecessary apply_terraform call
- Removed LocalStack dependency
- Uses hardcoded Docker connection strings instead
- Provides direct PostgreSQL, Redis, MinIO connection info
- Faster startup without AWS/Terraform overhead

## Technical Details

### Console Script Features
- **Multi-environment support**: local, staging, production
- **Container validation**: Checks if Docker containers are running
- **Security controls**: Production access requires explicit authorization
- **User guidance**: Interactive tips for common commands
- **Error handling**: Clear error messages with remediation steps

### Local Development Improvements
- Terraform completely optional for local development
- No AWS credentials required
- Direct Docker service access (PostgreSQL, Redis, MinIO)
- Connection strings provided by startup script
- Eliminates VPC creation errors for local environment

### Connection Strings (Local)
```

```

## Developer Experience

### New Available Commands
```bash
make local          # Start local environment (no AWS needed)
make db-console     # Access PostgreSQL console
make redis-console  # Access Redis CLI
make dev            # Start development server
```

### PostgreSQL Console Usage
```bash
make db-console

# Inside psql:
\l              # List databases
\dt             # List tables
\d table_name   # Describe table
\q              # Quit
```

### Redis Console Usage
```bash
make redis-console

# Inside redis-cli:
PING            # Test connection
KEYS *          # List keys
INFO            # Server info
exit            # Quit
```
