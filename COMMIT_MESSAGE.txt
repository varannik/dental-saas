feat(config): bootstrap @dental/config with validated env and DB client

Create a production-ready shared config package for Phase 1 foundation work so
services can consume strongly validated environment configuration and a reusable
Drizzle/Postgres database connection layer.

## What Changed

- Rewrote `packages/config/package.json` as a proper workspace package:
  - scoped name `@dental/config`
  - ESM exports + dist entry points
  - build/dev/typecheck scripts
  - runtime dependencies for env validation and DB access
- Added `packages/config/tsconfig.json` for package-local NodeNext compilation:
  - `rootDir`/`outDir`
  - declaration output and source maps
  - Node types enabled
- Added `packages/config/src/env.ts` with best-practice environment handling:
  - `.env` + `.env.local` loading
  - Zod schema validation with clear aggregated error output
  - empty-string normalization for optional secrets
  - cached accessor (`getEnv`) and exported singleton (`env`)
- Added `packages/config/src/database.ts` with a reusable Drizzle/Postgres layer:
  - typed `pg` pool creation
  - connection pooling defaults and overrides
  - singleton DB client access (`getDatabase`)
  - graceful shutdown helper (`closeDatabase`)

## Files Updated

- `packages/config/package.json`
- `packages/config/tsconfig.json` (new)
- `packages/config/src/env.ts` (new)
- `packages/config/src/database.ts` (new)
